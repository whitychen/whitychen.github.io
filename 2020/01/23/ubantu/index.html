Ubuntu 18.04.3 LTS 配置记录

2020年1月 张为

本篇无图且很简略，所以不适用于初学的教程，但是如果遇到一些问题，可以根据这里面的步骤、相关名词等信息查阅网上资料。

apt换清华源

现在默认好像是cn.archive.ubuntu.com了，并不太慢。如果觉得慢的话有国内源可用，清华tuna源的替换方式在这里。搞定之后可以更新一下预装软件包，通过sudo apt update && sudo apt upgrade。

咦，好像没有git？

Ubuntu两大生命之源：apt和git。怎么能没有git，我还要跑开源代码呢！赶紧sudo apt-get install git。

开始让系统可用 — 搞定显卡驱动和显示

无论是不是精简安装，应当都有火狐 (firefox) 浏览器，文件管理器 (files)，应用商店 (Ubuntu software)，和最重要的gnome终端 (terminal)。

如果是外置Nvidia显卡的话，现在可能会画质很差，分辨率很低，字体很差。无论是不是这样的情况，都最好安装一下Nvidia的驱动。如果画面正常的话，可能是预装的xerver上的nouveau起了作用，用apt list --installed | grep nouveau命令看一下，这里是有libdrm-nouveau2和xserver-xorg-video-nouveau-hwe-18.04两个东西。

再apt list --installed | grep nvidia看一下，确认没有N卡驱动，如果有的话用sudo apt purge nvidia*彻底删除一下并重新启动。如果图形界面彻底进不了先不要慌，Ctrl-Alt-F2进命令行，输入用户名和密码登录。

开始装显卡驱动。sudo lshw -c display能查看到一些显示信息。可以ubuntu-drivers devices查看识别了的硬件设备和对应的驱动。里面会有一个recommend的驱动，如果觉得可以就ubuntu-drivers autoinstall自动安装推荐驱动即可。想要从所有包里找的话就apt search --names-only nvidia看看现有的版本，一般不会有太大出入。如果觉得不够新可以使用第三方源，sudo add-apt-repository ppa:graphics-drivers，然后sudo apt update，然后再ubuntu-drivers devices或apt search --names-only nvidia，应当会有更新的版本。前者中推荐的驱动也可能会变成更新的，此时就可以自动或手动安装了。sudo shutdown -r now关机或者reboot重启。

此时应当有良好的显示了。用nvidia-smi查看，能看到比较易读的显卡信息。貌似是默认带最新CUDA了。

显示缩放

现在都是高分辨率屏，难免会字体过小，而Ubuntu 18.04的缩放直接从100%跳到200%……好吧我们来装一个工具，sudo apt install gnome-tweaks && gnome-tweaks，就能随便缩放啦。另外，再设置的Universal access里面可以调大一些的字体，和光标大小。注意调系统的设置可能会让tweaks的设置重置。

exFAT支持

U盘常见有exFAT格式，这一格式在Mac和Windows上都能读写，而且也没有FAT32的4GB的单文件限制，但是Ubuntu 18.04没有默认支持。需要sudo apt install exfat-fuse exfat-utils，再插上U盘应当能正常使用。

中文输入法

sudo apt install ibus-pinyin安装IBus拼音支持，或者喜欢别的就在apt里查一下。在设置—地区和语言 (region & language)里面，先点管理已安装语言 (manage installed languages)，可能会提示安装一些语言的周边支持包，建议安装以提供完整支持。

进入页面，下面有一个Keyboard input method，选择IBus，退出，重启。设置—地区和语言—输入源（Input sources），点加号，Chinese—Chinese (Intelligent Pinyin)，确认即可。

终端配色方案 —- solarized

第二步文件夹名称太长怎么办？多按Tab！默认的shell就有一定的自动补全了。运行安装脚本之后根据导引选择。Solarized还有一个主仓库在这里。

Vim

然后就需要一款好用的文本编辑器了，我习惯用Vim。下面是Vim+Vundle的安装过程。

先安装vim本体，建立bundle所有扩展包的文件夹，然后因为Vundle把自己也当成一个bundle，所以把Vundle本体拷进去。最后配置.vimrc文件，可以参照官方示例的格式。这就是个大坑了，如何找到好用的扩展包，如何配置扩展包，甚至自己写vim script（不会），本篇就不展开了，有需要的话我可以想办法发给你我的vimrc配置，用起来还比较顺手。

网络 —- 基于vmess协议的v2ray

危险操作，小孩子和不懂事的大人不要模仿（不是）

它有官网但显然是上不去的……经验告诉我们现在Github上搜，官方仓库在这里（居然不是排第一）。点进Releases，下载v2ray-linux-64.zip即可，解压即可用，不需要安装。设置config.json，然后log的地址都可以删掉，让它输出到屏幕就行，如果你是前台运行的话。在命令行里运行./v2ray -test检查设置是否ok，然后./v2ray运行。

相比SS/SSR我更推荐这个，因为现在SS查的比较严，即使用aes-256-gcm加密也不免在一段时间内被封（个人体验），这个目前来说还比较好用，可能是因为针对性开发的吧。它只会在持续大流量的时候被限制一段时间，解决方案就是先断一会儿再用。

插播广告 —- 如何通过ssh密钥安全地登录上服务器

挖坑。

然后关于设置，篇幅原因没法都放，我就用我一点也没有的网络知识大致讲一下思路吧。浏览器出来的请求正常情况是直接按照地址往外发送，然后如果设置了代理的话就会发送到127.0.0.1也就是本机的对应端口，v2ray在输入inbounds端监听了127.0.0.1的对应端口，然后根据设置的规则给到outbounds端，outbounds的代理这个选项把数据发送到服务器端的对应端口。服务器端的vmess输入端设置端口、所有用户的uuid，就能够接收，然后服务器端的输出就是自由的，直连。

同时v2ray也兼容ss的代理，所以服务器端可以设置两个端口，一个接收vmess输入，一个接收ss输入，这样在用户端也可以用ss来走代理，虽然这样是不建议的。

然后先设置全局代理。什么？你想用Proxy SwitchyOmega？但它在google商店里啊。什么？你的账号安装过所以可以同步？但是你要登录google账号啊。好，我们打开设置—网络—代理—手动，HTTP和HTTPS代理都设置成127.0.0.1的你在config.json里设置的输入端口。对了你可能还没有chromium，开源的chrome，到应用商店里下载就行了。试一下google应该ok，在运行v2ray的那个terminal上应该有信息的显示（别给关了，关了就断掉了）。然后先登录google账号（这样之后能在设备间同步），再找chrome://extensions，这里管理了你拥有的插件。点进google插件商店，下载Proxy SwitchyOmega。

设置SwitchyOmega。proxy一栏选HTTP（HTTPS应该也行），服务器是127.0.0.1，端口还是系统代理那个端口。auto switch里面删掉所有的示例，勾选rule list rules，URL输https://raw.githubusercontent.com/gfwlist/gfwlist/master/gfwlist.txt（就是github里面搜gfwlist，虽然github能上，但是github的raw是在墙外的……），点击下载。Format应当会自动切换到AutoProxy。上面调成匹配列表的就代理，不匹配的直连。至此，已经可以断掉系统代理了，别忘了断掉。调成自动切换模式，用google和baidu分别测试一下，有代理的就会有蓝圈。以后有网站上不去或很慢就点增加条件，让这个网站走代理。
